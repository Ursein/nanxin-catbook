# 南信猫友记——测试报告

## 1. 引言

### 1.1 编写目的
本文档旨在记录"南信猫友记"系统的软件测试过程和结果，包括单元测试、集成测试、确认测试和系统测试的测试用例设计、执行结果及缺陷分析。本文档的读者包括项目开发人员、测试人员及项目评审人员。

### 1.2 测试目标
1. 验证各模块功能是否符合需求规格说明书的要求
2. 验证模块间接口调用是否正确
3. 验证核心推荐算法（余弦相似度 + 热度因子融合）的计算准确性
4. 验证系统的性能、安全性和兼容性

### 1.3 项目背景
- **项目名称：** 南信猫友记（NanXin CatBook）
- **系统定位：** 校园流浪猫 Web 端信息管理平台，作为已有微信小程序的补充
- **测试范围：** 覆盖 10 个功能域、44 个测试用例，涉及 7 个 REST 控制器和 8 张数据库表的全链路验证
- **算法验证重点：** 余弦相似度计算、One-Hot 特征编码、Min-Max 归一化、多因子加权融合的全流程正确性
- **技术栈：** Spring Boot 3.4.5 + Vue 3 + MySQL 8

- 《南信猫友记——概要设计说明书》

## 6. 黑盒测试

### 6.1 测试方法

黑盒测试（Black-box Testing）将被测系统视为"黑盒子"，不关注内部实现，仅通过输入和输出验证功能正确性。本系统采用以下两种黑盒测试方法：

#### 6.1.1 等价类划分

将输入数据划分为有效等价类（可接受的值）和无效等价类（不可接受的值），每个等价类只需测试一个代表性样本。

| 测试项 | 有效等价类 | 无效等价类 | 说明 |
|--------|------------|------------|------|
| 评分输入（五维 r1-r5） | 1, 2, 3, 4, 5 | ≤0, ≥6 | 每一位评分取值 |
| 分页页码 page | 0, 1, 2, ... | -1, 负数 | 从 0 开始 |
| 分页条数 size | 1-100 | 0, 101+, -1 | 限制最大 100 |
| 猫咪名称长度 | 1-50 字符 | 空串, 51+ 字符 | varchar(50) |
| 关键词搜索 | 任意字符串 | SQL/JS 注入字符串 | 安全转义 |

#### 6.1.2 边界值分析

对输入范围的边界值及其相邻值进行测试，边界是软件缺陷的高发区域。
