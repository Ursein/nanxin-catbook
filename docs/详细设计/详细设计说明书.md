## 2. 总体设计

### 2.1 需求概述
本系统为校园流浪猫信息管理与推荐平台，提供猫咪档案管理、搜索浏览、照片审核、社交互动（点赞/关注/评分/留言）及基于余弦相似度的猫咪推荐功能。系统采用 B/S 架构，后端 Spring Boot 3.2.5 + Java 17，前端 Vue 3 + Vite，数据库 MySQL 8。

### 3.1 M1 用户管理模块

#### 3.1.1 功能
实现用户的注册、登录、个人信息管理以及三级角色权限控制（USER / VERIFIED / ADMIN）。

#### 3.1.2 性能
登录响应时间 ≤ 500ms，注册响应时间 ≤ 1s。

#### 3.1.3 输入项目
- 注册：用户名（username）、密码（password）、邮箱（email）
- 登录：用户名、密码
- 获取/更新个人信息：JWT Token

#### 3.1.4 输出项目
- 注册/登录成功：JWT Token + 用户基本信息
- 个人信息：用户 ID、用户名、昵称、邮箱、头像、角色

#### 3.1.5 算法
- 密码加密：BCrypt 哈希算法（`BCryptPasswordEncoder.encode()`）
- Token 生成：HMAC-SHA256 签名 JWT，有效期 24 小时

#### 3.1.6 程序逻辑
```
POST /auth/register
  1. 校验 username 唯一性
  2. 密码 BCrypt 加密
  3. 写入 user 表，角色默认 USER
  4. 返回成功消息

POST /auth/login
  1. 根据 username 查询用户
  2. BCrypt 验证密码
  3. 生成 JWT Token（含 userId + role）
  4. 返回 Token + 用户信息

GET /auth/me
  1. 从请求头提取 JWT Token
  2. 解析 Token 获取 userId
  3. 查询并返回用户信息
```

#### 3.1.7 接口
| 方法 | 路径 | 权限 |
|------|------|------|
| POST | `/api/v1/auth/register` | ALL |
| POST | `/api/v1/auth/login` | ALL |
| GET | `/api/v1/auth/me` | LOGIN |
| PUT | `/api/v1/auth/me` | LOGIN |

#### 3.1.8 测试要点
- 用户名重复注册应返回错误
- 错误密码登录应返回 401
- JWT Token 过期应返回 401
- 未携带 Token 访问需登录接口应返回 401

### 3.10 P6 核心算法详细描述

#### 3.10.1 功能
基于余弦相似度融合热度因子的内容推荐算法。在猫咪详情页，计算当前猫咪与其他所有猫咪的相似度，结合热度因子加权，推荐 Top-5 相似猫咪。

#### 3.10.2 性能
- 猫咪数量 ≤ 100 时，单次计算 ≤ 1s
- 特征向量预缓存，避免重复计算
- 热度因子每 15 分钟批量刷新

#### 3.10.3 输入项目
- 当前猫咪 ID
- 特征来源：cat 表的 colour_tags、personality_tags、location_area、gender
- 热度来源：photo_like 表（点赞数）、cat_follow 表（关注数）、cat_rating 表（评分均分）

#### 3.10.4 输出项目
相似猫咪 Top-5 列表，每项包含：猫咪 ID、名称、封面照片、综合推荐分数

#### 3.10.5 算法（伪代码）

```
算法：CatSimilarityAlgorithm
输入：当前猫咪ID (targetCatId)
输出：相似猫咪推荐列表 Top-N (默认 N=5)

Step 1 — 特征提取
  targetCat = SELECT * FROM cat WHERE id = targetCatId
  allCats = SELECT * FROM cat WHERE id != targetCatId AND status = 'ACTIVE'
  
  提取 targetCat 特征：
    colourTagsSet = targetCat.colour_tags.split(';')     // 毛色标签数组
    personalitySet = targetCat.personality_tags.split(';') // 性格标签数组
    location = targetCat.location_area                      // 位置
    gender = targetCat.gender                               // 性别

Step 2 — 构建全局特征词典
  globalColours = 所有猫咪 colour_tags 的并集
  globalPersonalities = 所有猫咪 personality_tags 的并集
  globalLocations = 所有猫咪 location_area 的并集
  totalDimensions = len(globalColours) + len(globalPersonalities) 
                    + len(globalLocations) + 3  // 3 = 性别维度数

Step 3 — 特征向量化（One-Hot 编码）
  FUNCTION encode(colourTags, personalityTags, location, gender):
    vector = [0] * totalDimensions
    offset = 0
    // 编码毛色
    FOR tag IN globalColours:
        IF tag IN colourTags: vector[offset] = 1
        offset++
    // 编码性格
    FOR tag IN globalPersonalities:
        IF tag IN personalityTags: vector[offset] = 1
        offset++
    // 编码位置
    FOR loc IN globalLocations:
        IF loc == location: vector[offset] = 1
        offset++
    // 编码性别 (MALE/FEMALE/UNKNOWN → 3维)
    genderMap = {'MALE': [1,0,0], 'FEMALE': [0,1,0], 'UNKNOWN': [0,0,1]}
    vector[offset:offset+3] = genderMap[gender]
    RETURN vector

  targetVector = encode(targetCat)
  catVectors = {}
  FOR cat IN allCats:
      catVectors[cat.id] = encode(cat)

Step 4 — 计算余弦相似度
  FUNCTION cosineSimilarity(vecA, vecB):
    dotProduct = SUM(vecA[i] * vecB[i] FOR i IN range(len(vecA)))
    normA = SQRT(SUM(vecA[i]^2 FOR i IN range(len(vecA))))
    normB = SQRT(SUM(vecB[i]^2 FOR i IN range(len(vecB))))
    IF normA == 0 OR normB == 0: RETURN 0
    RETURN dotProduct / (normA * normB)

  similarityScores = {}
  FOR catId, vector IN catVectors:
      similarityScores[catId] = cosineSimilarity(targetVector, vector)

Step 5 — 热度因子融合
  // 获取热度数据
  heatData = QUERY:
      SELECT 
          cat_id,
          like_count,       // 来自 photo_like 统计
          follow_count,     // 来自 cat_follow 统计
          avg_rating        // 来自 cat_rating 统计
      FROM cat_rating_stats  // 热度统计缓存表

  // Min-Max 归一化函数
  FUNCTION normalize(values):
    minVal = MIN(values)
    maxVal = MAX(values)
    IF maxVal == minVal: RETURN [0.5] * len(values)
    RETURN [(v - minVal) / (maxVal - minVal) FOR v IN values]

  likeNorm = normalize([heatData[catId].like_count FOR catId IN allCats])
  followNorm = normalize([heatData[catId].follow_count FOR catId IN allCats])
  ratingNorm = normalize([heatData[catId].avg_rating FOR catId IN allCats])

  // 权重配置（可调整）
  α = 0.7  // 余弦相似度权重
  β1 = 0.1 // 点赞热度权重
  β2 = 0.1 // 关注热度权重
  β3 = 0.1 // 评分热度权重

  finalScores = {}
  FOR catId IN allCats:
      finalScores[catId] = α * similarityScores[catId] 
                         + β1 * likeNorm[catId]
                         + β2 * followNorm[catId] 
                         + β3 * ratingNorm[catId]

Step 6 — 排序输出
  sortedCats = SORT allCats BY finalScores[catId] DESC
  RETURN sortedCats[0:N]  // 返回 Top-5
```

#### 3.10.6 程序逻辑（后端 Service 实现）

```
@Service
public class CatSimilarityAlgorithm {
    
    @Autowired private CatRepository catRepository;
    @Autowired private PhotoLikeRepository photoLikeRepository;
    @Autowired private CatFollowRepository catFollowRepository;
    @Autowired private CatRatingRepository catRatingRepository;
    
    public List<RecommendResult> recommend(Long targetCatId, int topN) {
        // 1. 获取目标猫咪和所有活跃猫咪
        Cat target = catRepository.findById(targetCatId);
        List<Cat> allCats = catRepository.findByStatus("ACTIVE");
        
        // 2. 构建特征词典
        FeatureDictionary dict = buildDictionary(allCats);
        
        // 3. 向量化目标猫咪
        double[] targetVector = encode(target, dict);
        
        // 4. 计算相似度
        Map<Long, Double> simScores = new HashMap<>();
        for (Cat cat : allCats) {
            if (cat.getId().equals(targetCatId)) continue;
            double[] vector = encode(cat, dict);
            simScores.put(cat.getId(), cosineSimilarity(targetVector, vector));
        }
        
        // 5. 融合热度因子
        Map<Long, Double> finalScores = fuseHeatFactors(simScores, allCats);
        
        // 6. 排序取 Top-N
        return finalScores.entrySet().stream()
            .sorted(Map.Entry.<Long, Double>comparingByValue().reversed())
            .limit(topN)
            .map(e -> new RecommendResult(e.getKey(), e.getValue()))
            .collect(Collectors.toList());
    }
}
```

#### 3.10.8 限制条件
- 算法仅适用于活跃猫咪（status = ACTIVE）
- 特征维度随猫咪数量增加，需控制词典大小
- 热度因子缓存每 15 分钟刷新，非实时数据

#### 3.10.9 测试要点
- 相同特征猫咪应获得高相似度分数
- 多热度因子加权后排序应合理
- 边界情况：仅有一只猫时返回空列表
- 性能测试：50+ 猫咪时计算时间 ≤ 1s
