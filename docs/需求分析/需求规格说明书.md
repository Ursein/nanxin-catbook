# 南信猫友记——需求规格说明书

## 1. 引言

### 1.1 编写目的
本文档旨在明确"南信猫友记"系统的功能需求和非功能需求，为后续的概要设计、详细设计和编码实现提供依据。本文档的读者包括项目开发人员、测试人员及项目评审人员。

### 1.2 项目背景
- **项目名称：** 南信猫友记（NanXin CatBook）
- **委托单位：** 南京信息工程大学 软件学院
- **开发单位：** 软件工程课程项目组
- **系统定位：** 一个面向校园流浪猫咪的信息管理与推荐系统，帮助师生了解和关爱校园猫咪

### 1.3 系统目标
1. 为校园猫咪建立电子档案，记录基本信息、照片和健康状况
2. 提供猫咪搜索和浏览功能，方便用户查找感兴趣的猫咪
3. **基于余弦相似度的推荐算法，在猫咪详情页推荐相似猫咪**
4. 支持用户参与猫咪档案的建设（添加猫咪、上传照片、留言评论、关注猫咪、评分点赞）
5. 为管理员提供数据管理权限，包括照片审核和评论审核
6. 提供照片审核流程，保证照片质量和内容安全
7. 引入猫友互动机制（点赞、评分、关注），增强用户参与度

### 1.4 参考资料
- 《软件工程导论》课程教材
- 程勇老师《命题文件》实验要求

## 2. 任务概述

### 2.1 运行环境
- **后端运行环境：** JDK 17 + MySQL 8.0
- **前端运行环境：** 现代浏览器（Chrome / Edge / Firefox）
- **开发框架：** Spring Boot 3.2 + Vue 3

### 2.2 条件与限制
1. 系统为 Web 应用，无需安装客户端
2. 用户需联网访问
3. 图片存储使用本地文件系统

## 3. 系统功能需求

### 3.1 功能划分

| 功能编号 | 功能模块 | 功能描述 |
|----------|----------|----------|
| F1 | 用户注册登录 | 用户注册账号、登录、退出登录；角色体系支持三级权限 |
| F2 | 猫咪信息管理 | 添加/编辑/删除猫咪档案（包含父母/好友关系字段） |
| F3 | 猫咪浏览搜索 | 浏览猫咪列表（支持状态 Tab 分类和毛色分类浏览）、按关键词搜索、按条件筛选、查看猫咪详情 |
| F4 | 猫咪详情展示 | 展示猫咪信息、照片、留言、评分、关注状态；展示相似猫咪推荐 |
| F5 | 照片管理 | 上传、审核、查看、删除照片；照片点赞 |
| F6 | 留言评论 | 发表、查看、删除留言 |
| F7 | 相似推荐（核心算法） | 基于余弦相似度计算猫咪之间的相似度，融合热度因子推荐相似猫咪 |
| F8 | 行为日志 | 记录用户浏览、搜索、点赞、评分等行为 |
| F9 | 关注猫咪 | 用户关注/取消关注猫咪，查看已关注列表 |
| F10 | 猫友评分 | 用户从五个维度（猫德、颜值、社交、干饭、活力）各给 1-5 星评分，统计平均分和评分人数 |

### 3.2 功能描述

#### F1 用户注册登录
| 项目 | 说明 |
|------|------|
| 输入 | 用户名、密码、邮箱（注册时） |
| 处理 | 注册时验证用户名唯一性，密码加密存储；登录时验证用户名密码并返回 JWT Token。用户分为三级角色：**USER**（普通用户，可浏览、收藏、评分、上传照片和留言）、**VERIFIED**（特邀用户，可信度高，照片无需审核或优先审核）、**ADMIN**（管理员，拥有所有管理权限） |
| 输出 | 注册结果 / 登录 Token + 用户信息 |

#### F2 猫咪信息管理
| 项目 | 说明 |
|------|------|
| 输入 | 猫咪名称、毛色、性格标签、位置、性别、绝育状态、父母ID、好友ID等 |
| 处理 | 新增猫咪写入数据库；编辑时更新对应字段；删除时级联删除照片和评论。支持记录猫咪的亲友关系（父亲、母亲、好友），在详情页展示关系图谱 |
| 输出 | 操作结果 |
| 权限 | 任何登录用户可添加；仅创建者和管理员可编辑/删除 |

#### F3 猫咪浏览搜索
| 项目 | 说明 |
|------|------|
| 输入 | 关键词、位置筛选、状态筛选、毛色分类 |
| 处理 | ① **状态 Tab 分类展示：** 首页猫咪列表按状态分 Tab 展示（在校/待领养/失踪/喵星），支持横向切换和触底分页加载；② **关键词搜索：** 按名称/昵称匹配；③ **条件筛选：** 按位置、状态、毛色分类等条件组合筛选；④ **毛色分类浏览：** 按毛色分类（狸花/奶牛/橘猫/纯色/玳瑁及三花等）聚合展示猫咪；⑤ 搜索结果按余弦相似度排序 |
| 输出 | 猫咪列表（含分页） |

#### F4 猫咪详情展示
| 项目 | 说明 |
|------|------|
| 输入 | 猫咪 ID |
| 处理 | 查询猫咪详细信息、照片列表、留言列表；调用算法引擎计算相似猫咪 |
| 输出 | 猫咪信息 + 照片 + 留言 + 相似猫咪推荐 Top-5 |

#### F5 照片管理
| 项目 | 说明 |
|------|------|
| 输入 | 图片文件、猫咪 ID |
| 处理 | **上传：** 用户上传照片后，状态为 PENDING（待审核）；管理员审核通过后状态变更为 APPROVED 并展示在前端，拒绝时填写拒绝原因；**删除：** 仅上传者或管理员可删除。**点赞：** 用户可对已审核通过的照片点赞/取消点赞，每张照片统计点赞数 |
| 输出 | 照片记录（含审核状态、点赞数） |
| 权限 | 登录用户可上传；管理员可审核；任何已登录用户可点赞 |

#### F6 留言评论
| 项目 | 说明 |
|------|------|
| 输入 | 猫咪 ID、留言内容 |
| 处理 | 写入留言表，关联用户和猫咪 |
| 输出 | 留言记录 |

#### F7 相似推荐（核心算法）
| 项目 | 说明 |
|------|------|
| 输入 | 猫咪 ID |
| 处理 | 提取猫咪特征（毛色标签、性格标签、位置、性别）→ one-hot 编码为向量 → 计算与所有其他猫咪的余弦相似度 → **融合热度因子**（点赞数、关注数、评分均分加权）得到综合推荐分数 → 按分数降序排列 |
| 输出 | 相似猫咪 Top-N 列表（含综合推荐分数） |
| 详见 | 详细设计说明书中的算法描述 |

#### F8 行为日志
| 项目 | 说明 |
|------|------|
| 输入 | 用户 ID、猫咪 ID、操作类型 |
| 处理 | 自动记录用户浏览详情页、搜索、点赞、评分、关注等行为 |
| 输出 | 日志记录（用于后续热度分析和推荐算法） |

#### F9 关注猫咪
| 项目 | 说明 |
|------|------|
| 输入 | 用户 ID、猫咪 ID |
| 处理 | 用户可关注/取消关注猫咪；关注后可在个人中心查看已关注猫咪列表；关注数作为热度因子纳入推荐算法 |
| 输出 | 关注状态 / 已关注猫咪列表 |
| 权限 | 任何已登录用户可操作 |

#### F10 猫友评分
| 项目 | 说明 |
|------|------|
| 输入 | 用户 ID、猫咪 ID、r1~r5（五个维度各 1-5 星） |
| 处理 | 用户可从猫德、颜值、社交、干饭、活力五个维度各给 1-5 星评分；每个用户对同一猫咪仅保留最新评分；系统统计猫咪的平均分和评分人数；评分均分作为特征维度纳入推荐算法 |
| 输出 | 评分结果 / 猫咪平均分与评分人数 |
| 权限 | 任何已登录用户可评分 |

## 4. 非功能需求

### 4.1 性能需求
- 页面响应时间不超过 2 秒
- 相似度计算响应时间不超过 1 秒（猫咪数量 ≤ 100 时）
- 支持至少 50 人同时访问

### 4.2 安全性需求
- 密码使用 BCrypt 加密存储
- API 请求使用 JWT Token 鉴权
- 数据库密码和 JWT 密钥不硬编码在代码中

### 4.3 可用性需求
- 界面简洁直观，操作路径不超过 3 步
- 猫咪详情页和信息编辑页布局清晰

### 4.4 可维护性
- 后端采用分层架构（Controller → Service → Repository）
- 算法模块独立封装，便于替换和升级
