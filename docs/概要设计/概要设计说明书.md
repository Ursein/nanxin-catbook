# 南信猫友记——概要设计说明书

## 1. 引言

### 1.1 编写目的
本文档旨在描述"南信猫友记"系统的概要设计。系统基于已有的微信小程序校园猫平台（中大猫谱）进行 Web 端扩展，新增推荐算法等核心功能。本文档从系统架构层面规划模块划分、数据库设计及接口设计，为后续详细设计与编码实现提供技术蓝图。系统规模涵盖 9 大功能模块、8 张数据库表、7 个 REST 控制器及 9 个前端页面。

### 1.2 适用范围
本文档适用于项目开发人员、测试人员及项目维护人员，作为后续详细设计的技术约束输入。

### 1.3 参考文档
- 《南信猫友记——需求规格说明书》

## 2. 模块结构设计

### 2.1 模块划分总图

```
南信猫友记系统
├── M1 用户管理模块
│   ├── M1.1 用户注册
│   ├── M1.2 用户登录（JWT 鉴权）
│   ├── M1.3 用户信息管理
│   └── M1.4 角色权限（USER / VERIFIED / ADMIN）
├── M2 猫咪管理模块
│   ├── M2.1 猫咪档案管理（增/删/改）
│   ├── M2.2 猫咪关系管理（父母/好友）
│   └── M2.3 猫咪状态管理（在校/待领养/失踪/离世）
├── M3 搜索浏览模块
│   ├── M3.1 猫咪列表展示（分页，支持状态 Tab 切换：在校/待领养/失踪/喵星）
│   ├── M3.2 关键词搜索
│   ├── M3.3 条件筛选（位置/状态/毛色分类）
│   ├── M3.4 毛色分类浏览
│   └── M3.5 猫咪详情展示
├── M4 照片管理模块
│   ├── M4.1 照片上传
│   ├── M4.2 照片审核（管理员）
│   ├── M4.3 照片删除
│   └── M4.4 照片点赞/取消
├── M5 评论管理模块
│   ├── M5.1 发表评论
│   ├── M5.2 查看评论
│   └── M5.3 删除评论
├── M6 算法推荐模块（核心）
│   ├── M6.1 特征提取（原始特征读取）
│   ├── M6.2 特征向量化（One-Hot 编码）
│   ├── M6.3 余弦相似度计算
│   ├── M6.4 热度因子融合
│   └── M6.5 排序输出（Top-N）
├── M7 关注管理模块
│   ├── M7.1 关注/取消关注
│   └── M7.2 已关注列表查询
├── M8 评分管理模块
│   ├── M8.1 提交评分
│   └── M8.2 评分统计
└── M9 日志管理模块
    ├── M9.1 行为日志记录
    └── M9.2 热度数据统计
```

| M4 照片管理 | M9 日志管理 | 记录点赞行为 |

| M7 关注管理 | M9 日志管理 | 记录关注行为 |

| M4 照片管理 | M2 猫咪管理 | 照片关联到猫咪档案 |

## 4. 接口设计

### 4.1 接口规范

- **基础路径：** `/api/v1`
- **认证方式：** JWT Token，放入 `Authorization: Bearer <token>` 请求头
- **响应格式：** JSON
  ```json
  {
    "code": 200,
    "message": "success",
    "data": { ... }
  }
  ```
- **分页参数：** `page`（从 1 开始，默认 1）、`size`（默认 10）
- **角色注释：** `[ALL]` 所有人可访问，`[LOGIN]` 登录用户，`[ADMIN]` 仅管理员

### 4.2 用户模块接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/auth/register` | 用户注册 | ALL |
| POST | `/auth/login` | 用户登录，返回 Token | ALL |
| GET | `/auth/me` | 获取当前用户信息 | LOGIN |
| PUT | `/auth/me` | 更新个人信息 | LOGIN |
| POST | `/auth/me/avatar` | 上传头像 | LOGIN |
| GET | `/user/cats` | 获取我创建的猫咪列表 | LOGIN |
| GET | `/user/ratings` | 获取我的评分记录 | LOGIN |

### 4.8 推荐算法接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/cats/{catId}/recommend` | 获取相似猫咪推荐 Top-5 | ALL |

**算法内部调用流程（不对外暴露）：**

```
GET /cats/{catId}/recommend
  → M6.1 特征提取（读取 cat.colour_tags, personality_tags, location_area, gender）
  → M6.2 One-Hot 编码为特征向量
  → M6.3 计算与所有其他猫咪的余弦相似度
  → M6.4 融合热度因子（点赞数、关注数、评分均分）
  → M6.5 排序取 Top-5，返回含综合推荐分数
```
