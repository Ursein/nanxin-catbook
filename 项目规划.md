## 算法方向（选定）

**基于余弦相似度的内容推荐算法（主方案）**  
**融合热度因子（点赞数、关注数、评分均分）**

### 算法原理

```
Step 1 — 特征编码
  每只猫咪特征（毛色、性别、性格、位置）编码为数值向量（one-hot encoding）

Step 2 — 向量化
  Cat_A → Vector_A = [v₁, v₂, v₃, ..., vₙ]
  Cat_B → Vector_B = [v₁, v₂, v₃, ..., vₙ]

Step 3 — 余弦相似度计算
  cos(θ) = (A · B) / (|A| · |B|)

Step 4 — 热度因子融合
  Score = α · cos(θ) + β₁ · LikeNorm + β₂ · FollowNorm + β₃ · RatingNorm
  权重默认：α=0.7, β₁=β₂=β₃=0.1

Step 5 — 排序输出
  按综合推荐分数降序排列，取 Top-N
```

### 热度因子说明

| 因子 | 含义 | 数据来源 | 权重 |
|------|------|----------|------|
| cos(θ) | 余弦相似度 | 猫咪特征向量 | 0.7 |
| LikeNorm | 点赞数归一化 | photo_like 表 | 0.1 |
| FollowNorm | 关注数归一化 | cat_follow 表 | 0.1 |
| RatingNorm | 评分均分归一化 | cat_rating 表 | 0.1 |

### 算法特点

| 特性 | 说明 |
|------|------|
| **有真实数学公式** | 余弦相似度是标准算法，报告可写公式推导 |
| **有数据处理** | 特征选择、编码方式、归一化、多因子加权 |
| **可视化好** | 可画出向量空间示意图、相似度矩阵、加权公式图示 |
| **扩展性强** | 搜索关键词同样向量化后计算相似度；热度因子可增可调 |
| **来源多样** | 热度数据来源涵盖 4 张表（activity_log, photo_like, cat_follow, cat_rating） |

## 实施计划

1. **概要设计** — 模块结构、数据库设计、接口设计 ✅ 已完成
2. **后端搭建** — Spring Boot 初始化、数据库建表、基础 CRUD

5. **功能联调** — 全流程贯通（含照片审核、点赞、关注、评分）
6. **课程文档** — 详细设计 + 测试报告

## 竞品借鉴

本系统参考了 **中大猫谱（笃行猫谱）**（github.com/sysucats/zhongdamaopu）的以下设计：

| 借鉴项 | 实现方式 |
|--------|----------|
| 照片审核流程 | PENDING→APPROVED/REJECTED，管理员审核 |
| 照片点赞 | 点赞表 + 点赞数冗余字段 |
| 关注猫咪 | 关注表 + 已关注列表 |
| 猫友评分 | 评分表 + 平均分统计 |
| 三级权限 | USER / VERIFIED / ADMIN |

这些功能不仅提升了系统的完整性和用户参与度，还直接为推荐算法提供了热度数据源。
